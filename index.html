Docs and Integration

Use Open SDK contracts

DancingMusic keeps SDK contracts separate from the app host. Music connectors implement the MusicStore contract, while Dance visual plugins implement the DancingPluginSdk lifecycle.

  • Keep implementation repositories independent.
  • Build ESM bundles that can be loaded dynamically.
  • Pin production URLs to version tags instead of main branches.
Browse SDK repositories

Publish a music connector

A connector should expose search, playlist, stream URL, and optional lyrics capabilities through the public MusicStore interface.

  • Ship a `dist/index.js` ESM bundle.
  • Add contract tests for search and stream resolution.
  • Provide a docs demo page for quick validation.
View connector examples

Build a Dance plugin

Dance plugins receive audio and playback context from the app, then render visual stages through the plugin lifecycle. The built-in Night Sky stage follows the same host-facing plugin boundary.

  • Keep rendering code inside the plugin.
  • Use the shared audio analysis contract.
  • Expose metadata so the app can list and activate the plugin.
Explore plugin SDK

Verify downloads

Public packages are attached to the DancingMusic main repository releases. Use the generated metadata to verify version, build identity, and release status.

  • Latest metadata is available at `releases.json`.
  • Downloads are published as main repository release assets.
  • Source release URLs are not used as public links.
